# Milestone M7: Moderation, Safety & Guardrails

**Status:** COMPLETE  
**Completion Date:** 2026-05-24  
**Objective:** Establish deterministic, observable guardrails that prevent the system from violating platform constraints, avoid bot detection, and enforce a human-in-the-loop review process.

## Executive Summary
This milestone focused on defensive engineering. We implemented platform-specific limit validation, improved human-like browser behavior, prevented duplicate posting via historical tracking, and added a mandatory human-review step in the dashboard before content is published.

## Key Deliverables

### 1. Moderation & Content Guardrails
*   **Moderation Adapter:** Created `ModerationAdapter` (`src/autonomedia/ai/moderation/adapter.py`) to validate AI-generated text against hard platform constraints (e.g., character limits: 500 for Mastodon) before persisting the state.
*   **Multi-Platform Target Support:** Upgraded database schema (`platforms` JSONB array) and dashboard UI to allow selecting target platforms (Mastodon, X, LinkedIn) for a single idea.
*   **Targeted AI Generation:** Adjusted the `PostingSecretary` to prompt the AI individually per targeted platform and validate the output per platform's rules.

### 2. Browser Profiling & Anti-Detection
*   **Randomized Viewports:** Updated `BrowserProvider` to randomize browser window sizes on launch, reducing predictable fingerprinting.
*   **Human Cadence:** Replaced static timeouts with randomized `asyncio.sleep(random.uniform(0.5, 2.5))` delays in platform handlers to mimic human interaction speeds.
*   **Session Health Checks:** Implemented proactive checks for "Log in" / "Sign in" elements in platform adapters. The system now fails fast if browser cookies have expired, rather than waiting for timeouts on missing DOM elements.

### 3. State Machine & Execution Safety
*   **Duplicate Prevention:** Created a `post_history` database table with a `UNIQUE (content_id, platform)` constraint. The Executor checks this table before posting to prevent duplicate runs across worker restarts.
*   **Human-in-the-Loop Review:** Altered the state machine (`idea -> approved -> prepared -> ready_to_post -> published`). Content generated by AI now waits in the `prepared` state. Users must explicitly review and approve the text in the Dashboard before it queues for publishing.
*   **Deterministic Post Validation:** Executor verifies that the target platform's "Post" button is not disabled before clicking, catching subtle platform rule violations.

## Architectural Notes
*   **Safety Snapshot:** Adhered to the `Read-First` and `Pre-flight Snapshot` constraints for modifications.
*   **No Universal Logic:** Browser handlers remain strictly isolated per platform (e.g., `mastodon/task_handler.py`), avoiding fragile "universal posting" abstractions.
