# Milestone M6: AI Rewrite Layer & Assembly Line

**Status:** COMPLETE  
**Completion Date:** 2026-05-23  
**Objective:** Establish reliable stateless transformation and state-driven workflow for autonomous publishing.

## Executive Summary
This milestone transformed the publishing pipeline from a simple script into a robust, state-driven "Assembly Line." The system now separates AI transformation (The Secretary) from browser execution (The Executor), connected via a persisted database state.

## Key Deliverables

### 1. AI Transformation Layer
*   **Rewrite Adapter:** Implemented `RewriteProvider` interface and `GeminiProvider` using the new `google-genai` SDK.
*   **Reliability:** Implemented a robust fallback model chain (`gemini-2.5-flash`, `gemini-2.0-flash-lite`, etc.) to automatically handle 404/429 errors.
*   **Verification:** Established "Golden Master" testing (`tests/ai/test_rewrite.py`) to monitor AI output consistency.

### 2. State-Driven Workflow (The Assembly Line)
*   **Database Migration:** Applied schema changes for `prepared_content`, `scheduled_at`, and `error_log` to support content staging.
*   **State Machine:** Implemented status transitions (`idea` -> `approved` -> `prepared` -> `published`) to decouple logic.
*   **Background Workers:**
    *   **Secretary (`PostingSecretary`):** Async worker that polls for `approved` items, transforms them via AI, and promotes to `prepared`.
    *   **Executor (`PostingExecutor`):** Async worker that picks up `prepared` items and handles browser-based publishing.

### 3. Dashboard UX
*   **Add Content:** Form now requires essential fields (`topic`, `link`, `idea`).
*   **Deletion:** Added browser-native confirmation (`hx-confirm`) to prevent accidental data loss.
*   **Parsing:** Refactored hash/mention handling to permit raw user input, supporting custom formats.

## Architectural Notes
*   **Observability:** Integrated `structlog` for JSON-formatted operational logging across all workers.
*   **Safety:** Credentials managed strictly through `.env` via `python-dotenv`.
*   **Determinism:** Browser interactions utilize `playwright` with explicit verification steps to confirm posts appear live.
