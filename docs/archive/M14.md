# Milestone M14: Daily Posting Routine with Randomized Intervals

## Goal
Implement a manual-trigger daily posting routine that processes verified Mastodon content from the queue, applies randomized delays between platforms, and ensures no verified content is left behind through full iteration with expiration awareness.

## Scope
- Backend: Add `posting_routine()` function to batch process verified Mastodon content
- Backend: Query `ready_to_post` items with `verification_status.mastodon = true`
- Backend: Implement randomized delay system (2-10 min between platforms)
- Backend: Post 1-2 items per run, allowing morning/afternoon style posting
- Backend: Add expiration timestamp to verified content (TTL for priority posting)
- Backend: Track last_posted_at per platform for intelligent scheduling
- CLI/Script: Create `scripts/checks/run_daily_posting.py` for manual invocation
- Verification: Test posting flow WITHOUT actual online posting (dry-run mode)
- Verification: Test expiration priority ordering for recent content

## Out of Scope
- Auto-scheduling or daemon mode (manual trigger only)
- Multi-platform expansion (X, LinkedIn) - Mastodon only
- Engagement feedback loops (likes, reshares)
- Complex rate-limit management beyond simple random delays
- VPS deployment (future milestone)

## Completion Criteria
- [ ] `posting_routine()` function exists and processes verified Mastodon items
- [ ] Randomized delays (2-10 minutes) applied between platform batches
- [ ] 1-2 items posted per daily run with catchup capability
- [ ] Expiration timestamp field added to track content freshness
- [ ] Last-posted tracking per platform prevents duplicate timing
- [ ] Manual trigger script runs successfully without errors
- [ ] Dry-run mode verified for testing without real online posts
