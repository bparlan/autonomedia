**File:** `docs/verifications/M13S1V.md`

## Success Conditions
- The `verification_status` column exists in the `content` table after migration.
- The column type is `JSONB` with a default value of an empty object `{}`.
- Existing rows retain all original data (no loss of non‑verification fields).
- The migration script (`scripts/db/migrate_M13_verification_status.py`) runs idempotently (`ADD COLUMN IF NOT EXISTS`).
- Running the migration multiple times produces no errors and leaves the schema unchanged after the first successful run.

## Edge Cases
- **Large JSON payload** – the column should accept a JSON object with many keys (e.g., 100 platforms) without error.
- **Malformed JSON input** – attempts to write non‑JSON data must be rejected by PostgreSQL and raise an exception.
- **Concurrent updates** – simultaneous updates to the same `verification_status` field should not cause deadlocks; the DB’s row‑level locking must handle it.
- **Network/DB outage during migration** – if the migration is interrupted, the schema remains unchanged (no partial column creation).

## Failure Conditions
- Column missing or wrong type after migration.
- Default value is `NULL` instead of `{}`.
- Migration script crashes and leaves the DB in a partially‑migrated state.
- Existing rows lose data unrelated to `verification_status`.

## Manual Validation Steps
1. Run the migration script against a fresh test database.
2. Inspect the `content` table schema (`\d content`) to verify column definition.
3. Insert a row without specifying `verification_status` and ensure the default `{}` is stored.
4. Update the column with a large JSON object and confirm it persists.
5. Simulate a failed migration (e.g., kill the process mid‑run) and verify the column is not created.

## Regression Risks
- Adding a JSONB column may affect any raw SQL queries that select `*` from `content`; ensure they still work with the new column.
- Existing ORM models (`src/database/models.py`) must be updated to include the new field, otherwise attribute errors may appear at runtime.
- Any bulk‑insert scripts that explicitly list columns need to add `verification_status` or rely on the default.
