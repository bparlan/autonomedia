# Verification: M13S4-AI Rewrite Dashboard Verified State

## Success Conditions
- Platform Status column displays a "Verified - In Queue" badge for items with `ready_to_post` status.
- For items targeting multiple platforms, each verified platform shows a distinct green badge indicating verification.
- Non‑verified platforms retain their existing status indicators.
- The Review/Verify link remains present and functional for all `ready_to_post` items.

## Edge Cases
- Item with `ready_to_post` but no `verification_status` data should fallback to a generic "Verified - In Queue" badge.
- Items with a mix of verified and non‑verified platforms display both badge types correctly.
- Items where `verification_status` JSON is malformed should not break the UI; the badge should be omitted gracefully.

## Failure Conditions
- Platform Status column shows no badge or an incorrect badge for any `ready_to_post` item.
- Verified platforms are not visually distinguished (e.g., missing green styling).
- Review/Verify link disappears or becomes non‑interactive for `ready_to_post` items.
- UI throws errors when rendering malformed `verification_status` data.

## Manual Validation Steps
1. Navigate to the AI Rewrite Dashboard (`/rewrites`).
2. Locate items with status `ready_to_post`.
3. Verify that a "Verified - In Queue" badge appears.
4. For multi‑platform items, confirm each verified platform shows a green badge.
5. Click the Review/Verify link to ensure it opens the editing interface.
6. Introduce a deliberately malformed `verification_status` JSON via the backend and confirm the UI handles it without crashing.

## Test Cases
- **TC1:** Single‑platform `ready_to_post` item → badge appears.
- **TC2:** Multi‑platform `ready_to_post` item with some platforms verified → correct badges per platform.
- **TC3:** `ready_to_post` item with empty `verification_status` → generic badge displayed.
- **TC4:** Item with malformed `verification_status` JSON → UI remains stable, no badge.
- **TC5:** Review/Verify link click on any `ready_to_post` item → editing page loads.

## Regression Risks
- Changes to `partials/content_row.html` may affect other status columns if conditional logic is too broad.
- Introduction of new CSS classes for badges could clash with existing styles.
- Backend changes to `verification_status` handling might impact other dashboard components.
