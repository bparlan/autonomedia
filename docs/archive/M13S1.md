# Spec: M13S1 - Database Schema Migration

## Problem

The `content` table lacks the `verification_status` JSONB column needed to track per-platform verification states. While M12's completion notes mention this column was added, current schema.py does not include it in the INIT_SCHEMA constant. This column is essential for M13's per-platform visibility and verification tracking.

## Requirements

PostgreSQL JSONB column stores per-platform verification metadata with nested structure:
```json
{"platform_name": {"verified": true/false, "verified_at": "ISO-timestamp", "expires_at": "ISO-timestamp"}}
```

This structure supports M12's granular verification and enables M13's per-platform visibility. The empty default `{}` ensures backward compatibility with existing items.

## Architecture

PostgreSQL JSONB column stores per-platform verification metadata. Migration script uses `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` pattern for idempotency.

## Implementation Plan

1. Create `scripts/db/migrate_M13_verification_status.py` with asyncpg migration
2. Add `verification_status JSONB DEFAULT '{}'::jsonb` to `src/database/schema.py` INIT_SCHEMA
3. Execute migration in development environment
4. Verify column exists and has correct default value
