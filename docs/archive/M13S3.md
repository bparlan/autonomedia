# Spec: M13S3 - Frontend Ready to Publish Per-Platform Rows

## Problem

The "Ready to Publish" section in Command Center shows aggregated items (one row per content ID), but operators need per-platform visibility to see which platform each prepared content targets. This granular view is required for operational oversight before publishing.

## Requirements

- Display one row per platform-item combination in Ready to Publish section
- Format: `#ID - Topic | Platform | Content Preview (100 chars) | Status`
- Content preview displays first 100 characters in small font (Tailwind `text-xs`)
- Status shows "Verified - In Queue" badge for verified platforms
- Platform badge color-coded (green for verified)
- Unqueue action only available via Review/Verify link (minimal Command Center)

## Architecture

Jinja2 template iteration over `prepared_content` JSONB keys. Each platform gets its own `<tr>` in the dashboard table. Template receives `item` with `prepared_content` parsed, iterates platforms to render individual rows.

## Implementation Plan

1. Modify `src/web/templates/dashboard.html` Ready to Publish section
2. Add outer loop over `prepared_content` items to create per-platform rows
3. Render columns: ID/Topic (combined), Platform name, Content Preview (100 chars truncated), Status badge
4. Add conditional badge styling: green background for verified platforms
5. Ensure Review/Verify link remains visible for all rows

## Verification Strategy

Link to: `.docs/verifications/M13S3V.md` (to be created via `pi verify M13S3`)

## Acceptance Criteria

- Each platform shows as separate row in Ready to Publish table
- Content preview displays first 100 characters in `text-xs` styling
- Verified platforms show "Verified - In Queue" badge with green styling
- Non-healthy platforms show distinct visual treatment (grayed out or warning)
- Row count equals sum of all platforms across ready_to_post items

## Dependencies

- M13S1 (Database schema with verification_status column)
- M13S2 (Backend endpoint modifications)
