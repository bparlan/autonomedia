# Spec: M13S4 - AI Rewrite Dashboard Verified State

## Problem

The AI Rewrite Dashboard (`/rewrites`) needs to distinguish verified `ready_to_post` items from other items and show their verification state. Currently the Platform Status column shows "Queued" but doesn't indicate which specific platforms are verified for a multi-platform item.

## Requirements

- "Verified - In Queue" badge displays in Platform Status column for `ready_to_post` items
- Badge shows per-platform verification status when multiple platforms exist
- Review/Verify link remains accessible for editing verified content
- Status column reflects current verification state per platform

## Architecture

Update `partials/content_row.html` to check `verification_status` JSON and render appropriate badge. For single-platform items, show "Verified - In Queue". For multi-platform items, show platform badges indicating which are verified.

## Implementation Plan

1. Update `src/web/templates/partials/content_row.html` Platform Status `<td>`
2. Add conditional rendering for `ready_to_post` status
3. Check `verification_status` JSON to determine verified platforms
4. Render "Verified - In Queue" badge or per-platform verification indicators
5. Ensure Review link remains available for `ready_to_post` items (allowing edits)

## Verification Strategy

Link to: `docs/verifications/M13S4V.md` (to be created via `pi verify M13S4`)

## Acceptance Criteria

- `ready_to_post` items show "Verified - In Queue" badge in Platform Status column
- Verified platforms within an item are visually distinct (green badges)
- Non-verified platforms show appropriate status (if any exist)
- Review/Verify link remains available for all `ready_to_post` items

## Dependencies

- M13S1 (Database schema with verification_status column)
- M13S2 (Backend endpoint modifications)
