# Verification: M13S3-ReadyToPublish-PerPlatformRows

## Success Conditions
- The Ready to Publish table displays a separate row for each platform-item combination.
- Content preview column shows the first 100 characters of the content, rendered with Tailwind `text-xs` styling.
- Verified platforms display a green "Verified - In Queue" badge.
- Non‑verified platforms display a distinct visual cue (e.g., gray badge or warning style).
- The total number of rows equals the sum of all platforms across all ready‑to‑post items.

## Edge Cases
- Items with no `prepared_content` entries should not render any rows.
- Content shorter than 100 characters should display the full text without truncation.
- Platforms with unknown verification status should default to a neutral styling.
- Extremely long topic strings should truncate gracefully to avoid breaking table layout.

## Failure Conditions
- Multiple platforms are collapsed into a single row.
- Content preview exceeds 100 characters or lacks `text-xs` styling.
- Badge colors do not reflect verification status.
- Missing Review/Verify links for any platform row.

## Manual Validation Steps
1. Navigate to the Command Center dashboard.
2. Populate the system with test data covering:
   - Multiple items each with several platforms (verified & non‑verified).
   - Items with a single platform.
   - Items with no platforms.
3. Verify that the table renders per‑platform rows as specified.
4. Inspect badge colors and content preview length.
5. Click the Review/Verify link for a few rows to ensure it functions.

## Regression Risks
- Changes to the dashboard rendering loop could unintentionally re‑introduce collapsed rows.
- Modifications to Tailwind classes may affect badge styling.
- Backend changes to `prepared_content` structure could break iteration logic.
