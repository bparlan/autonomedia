# Spec: M14S1 - Daily Posting Routine with Randomized Intervals

## Problem

Currently, the `PostingWorker` in `posting_executor.py` runs as an infinite daemon without:
1. Verification status awareness (posts content regardless of platform verification state)
2. Randomized delays between platform batches (immediate posting lacks natural pacing)
3. Expiration-aware content prioritization (older verified content loses priority over time)
4. Manual trigger capability (daemon mode only, no on-demand execution)

This prevents safe, controlled daily posting with human-like timing patterns.

## Requirements

- Backend: Query `ready_to_post` items where `verification_status.mastodon.verified = true`
- Backend: Implement randomized delay system (2-10 minutes between platform batches)
- Backend: Post 1-2 items per run with configurable limits
- Backend: Add `verified_at` and `expires_at` to `verification_status.mastodon` structure
- Backend: Prioritize content by `verified_at` (recent first), with expiration awareness
- Backend: Track `last_posted_at` per platform via `post_history.created_at` query
- CLI/Script: Create `scripts/checks/run_daily_posting.py` for manual invocation
- Verification: Dry-run mode for testing without actual online posting
- Verification: Expiration priority ordering validated via test scenarios

## Architecture

**module**: `src/autonomedia/core/posting_routine.py`

```python
# verification_status structure (extended)
{
  "mastodon": {
    "verified": true,
    "verified_at": "2026-06-16T10:00:00Z",
    "expires_at": "2026-06-16T20:00:00Z"  # Optional: TTL from verified_at
  }
}
```

Posting flow:
1. Query `content` WHERE status='ready_to_post' AND `verification_status->platform->>'verified'='true'`
2. Sort by `verification_status->platform->>'verified_at'` descending (recent first)
3. Apply randomized 2-10 minute delay between platform batches
4. Limit to 1-2 items per execution
5. Support `dry_run=True` to skip actual browser publishing
6. Update `post_history` on successful posting

## Implementation Plan

1. Create `src/autonomedia/core/posting_routine.py` with `posting_routine(dry_run: bool = False, max_items: int = 2)`
2. Add expiration calculation in verification logic (set `expires_at` 12 hours after `verified_at`)
3. Implement `_get_verified_content(platform: str)` query with verification status filter
4. Implement `_apply_randomized_delay(min_minutes: int, max_minutes: int)` with asyncio.sleep

6. Create `scripts/checks/run_daily_posting.py` CLI wrapper
